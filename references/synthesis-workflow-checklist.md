# Synthesis Workflow Checklist

Use this checklist while running the skill.

## Input Checks

- [ ] Assessment directory exists
- [ ] Empty placeholder files are excluded
- [ ] Each usable report contains `Synthesis Input Summary`
- [ ] Repository names extracted consistently
- [ ] Dimension ratings extracted consistently

## Normalization Checks

- [ ] Each report has a normalized summary
- [ ] Risk tags are engineering concepts, not exact phrases only
- [ ] Candidate standards are not copied blindly from a single report
- [ ] Learning signals are tied to evidence

## Synthesis Checks

- [ ] Themes are grouped by engineering concept
- [ ] Isolated severe risks are separated from systemic patterns
- [ ] Standards candidates are recurring, consequential, teachable, and actionable
- [ ] Every standards candidate includes problem, preferred, and testing code examples
- [ ] Learning recommendations are supportive and practical
- [ ] Every learning recommendation includes a before/after code example
- [ ] Executive summary frames reports as diagnostic evidence, not standards

## Final Review Checks

- [ ] No individual developers are named
- [ ] No raw repository report is included unless explicitly requested
- [ ] No conclusion is unsupported by assessment evidence
- [ ] The output helps the team decide what to standardize, learn, or remediate
